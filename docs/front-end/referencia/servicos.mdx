# Serviços

Os serviços são registrados com `providedIn: 'root'` (singleton em toda a aplicação) e centralizam a comunicação com a API REST.

## AuthService

**Arquivo:** `src/app/services/auth.service.ts`

**Base URL:** `environment.apiUrl`

| Método | HTTP | Endpoint | Auth |
|---|---|---|---|
| `cadastrar(dto)` | POST | `/usuarios` | Não |
| `login(email, senha)` | POST | `/usuarios/login` | Não |
| `getUserData()` | GET | `/usuarios/me` | Bearer |
| `saveToken(token)` | — | localStorage | — |
| `getToken()` | — | localStorage | — |
| `logout()` | — | localStorage | — |

### DTO de cadastro

```typescript
{
  nome: string;
  email: string;
  senha: string;
}
```

### Resposta de login

```typescript
{
  token: string;
}
```

## PlanoService

**Arquivo:** `src/app/services/plano-service.service.ts`

**Base URL:** `{apiUrl}/planos-aula`

Todos os métodos requerem autenticação (header `Authorization`).

| Método | HTTP | Endpoint | Descrição |
|---|---|---|---|
| `criar(dto)` | POST | `/planos-aula` | Cria novo plano |
| `atualizar(id, dto)` | PUT | `/planos-aula/{id}` | Atualiza plano existente |
| `obterPorId(id)` | GET | `/planos-aula/{id}` | Busca plano por ID |
| `listarPorDisciplina(disciplina)` | GET | `/planos-aula/disciplina/{disciplina}` | Filtra por disciplina |
| `listarMeusPlanos()` | GET | `/planos-aula/meus-planos` | Lista planos do usuário logado |
| `listarHabilidades()` | GET | `/plano-habilidade-bncc` | Lista habilidades vinculadas |
| `deletar(id)` | DELETE | `/planos-aula/{id}` | Remove plano |

### DTO de plano de aula

```typescript
{
  serie: string;
  disciplina: string;
  data?: string;
  duracao?: number;
  tema?: string;
  conteudo?: string;
  objetivos?: string;
  materiais?: string;
  metodologia?: string;
  avaliacao?: string;
  referencia?: string;
  habilidadesBnccIds: string[];
}
```

### Resposta de plano (leitura)

```typescript
{
  id: string;
  serie: string;
  disciplina: string;
  data: string;
  duracao: number;
  tema: string;
  conteudo: string;
  objetivos: string;
  materiais: string;
  metodologia: string;
  avaliacao: string;
  referencia: string;
  habilidades?: Array<{ id: string; descricao: string; codigo?: string }>;
  habilidadesBncc?: Array<{ id: string; descricao: string; codigo?: string }>;
}
```

## BnccService

**Arquivo:** `src/app/services/bncc.service.ts`

**Base URL:** `{apiUrl}/habilidades-bncc`

| Método | HTTP | Endpoint | Descrição |
|---|---|---|---|
| `buscarHabilidadesPorTermo(termo)` | GET | `/habilidades-bncc/buscar?termo={termo}` | Autocomplete BNCC |

### Resposta

```typescript
Array<{
  codigo: string;
  descricao: string;
}>
```

### Uso no componente

O `PlanoAulaComponent` consome este serviço com operadores RxJS:

```typescript
this.buscaBnccControl.valueChanges.pipe(
  debounceTime(400),
  distinctUntilChanged(),
  switchMap(termo => {
    if (!termo || termo.length < 3) return of([]);
    return this.bnccService.buscarHabilidadesPorTermo(termo);
  })
).subscribe(resultados => this.sugestoesBncc = resultados);
```

## Padrão de headers autenticados

```typescript
private getHeaders(): HttpHeaders {
  const token = this.authService.getToken();
  return new HttpHeaders().set('Authorization', `Bearer ${token}`);
}
```

Este padrão é replicado em `PlanoService` e `BnccService`. Não há interceptor HTTP centralizado.
